可复用的两步法游戏UI设计流程模板
Executive Summary
如果目标是把“原型图直接变成可用的游戏 UI”，同时还能稳定支持风格迁移、页面延展和组件拆解，那么最稳的做法不是把需求一次性塞进一条长提示词，而是把流程拆成两步：先拆“功能结构”，再生“视觉风格”。这套拆法与主流设计系统和游戏 UI 工程实践是同向的：Apple 近年的 HIG 将层级、和谐与一致性作为核心原则；Material 强调用颜色、尺寸、间距、位置和容器关系建立强视觉层级；Fluent 用留白、接近性和对比度来帮助用户快速判断主次；Unity 与 Unreal 的官方 UI 文档则都把“可复用控件/模板 + 样式覆盖”作为规模化生产 UI 的基础。基于这些共识，一个真正可复用的“设计 Skill”应当输出四类稳定产物：功能拆解表、风格提取表、最终生图提示词、以及可直接给 AI 使用的参数块。这样既能保住页面功能不丢失，也能避免把参考图“照抄成另一张图”。 

方法基线与适用范围
这套流程的核心是把结构与样式解耦。前者回答“这个页面是做什么的、有哪些模块、玩家要点哪里、页面有哪些状态”，后者回答“这个页面看起来像哪一类游戏、色彩和材质是什么、按钮和图标如何成立、字体和角色气质怎样统一”。Material 与 Fluent 都用 design tokens 管颜色、字体、间距和形状，避免把风格写死在单个页面里；Unity 的 USS/UXML 和 Unreal 的 Widget Template 也都在强调同一件事：结构要复用，样式要可覆盖，实例要可延展。把这两个维度分开，提示词稳定性会明显提高。

为了让生成图更接近“可交付 UI”而不是“好看的插画”，建议把下面这些技术约束写进模板：文字必须可读、按钮必须可点、对比要够、分辨率要高、留白要能承受多语言扩张。Apple 建议触控控件至少 44×44 pt、正文至少 11 pt，并要求高分辨率素材；Material 建议大多数平台的触控目标至少 48×48 dp；Fluent 要求标准文本对比度至少 4.5:1，图标和交互组件至少 3:1；Apple 还建议在更大字号、长文案和不同语言下测试重排，避免截断和重叠。

原型图功能拆解
第一步的目标不是“描述这张图长什么样”，而是把它翻译成一个页面功能模型。Material 明确提出，视觉层级应通过颜色、尺寸、间距、位置和容器关系来强调重点；Fluent 进一步说明留白和接近性会改变用户对信息关系与优先级的判断；Apple 则强调组织、对齐、可扫描性和把控件放在其所作用内容附近。换句话说，原型图拆解时最先要保住的是用户任务链路，不是美术元素。 

建议的拆解顺序
拆解项	要回答的问题	产出格式
页面类型	这是商店、背包、任务、抽卡还是设置页？	单句定义
核心任务	玩家进入页面后第一目标是什么？	单句动词短语
信息层级	哪些信息必须先看见？哪些是次要？	主/次/辅助三级
模块清单	顶栏、分类、列表、预览、奖励、关闭是否齐全？	模块表
主操作	页面里最重要的 CTA 是什么？	主按钮定义
状态字段	倒计时、资源、进度、选中项、未解锁等是否存在？	状态表
组件清单	哪些东西未来要复用？	组件命名表
文案约束	哪些文案必须保留？哪些可重写？	必保留/可改写表
图层边界	背景、面板、卡片、按钮、装饰是否分层明确？	图层说明

功能拆解检查清单
检查项	通过标准
页面类型明确	名称唯一，不与其他页面混淆
核心任务明确	能用一句话说清玩家来这里做什么
主按钮唯一	页面内只有一个最强 CTA
模块完整	资源、导航、主体、状态、退出至少齐全
状态齐全	选中态、禁用态、倒计时、进度、异常态被记录
数据可追溯	所有数值、价格、货币、进度来源明确
文案可落地	关键标题和数字已确定语言与格式
组件可复用	按钮、卡片、资源图标、进度条被单独命名
多语言可扩展	预留中文/英文长度变化的空间
移动端可点击	主交互区域足够大，便于触控

其中，关于“多语言可扩展”和“移动端可点击”的标准最好用平台规范校验：Apple 建议测试更大字号、避免文字重叠或不可用截断，并且在长单词语言、RTL 语言下检查布局；Apple 的最小命中目标是 44 pt，Material 的推荐触控目标通常是 48 dp，Fluent 也要求用间距来确保可感知与可点击。 

风格提取与提示词生成
第二步不是“复刻参考图”，而是提炼参考图的风格 DNA。Material 和 Fluent 都把 design tokens 当成跨设计与开发的共同语言，用来承接颜色、字体、间距、形状、阴影与高程；Unity 用 USS 与自定义控件管理外观与复用；Unreal 则允许在实例级别覆盖 Widget 的样式和行为。这意味着，在提示词层面最稳定的写法不是“像这张图一样”，而是把风格拆成可描述、可替换、可覆盖的字段。

对于“高质量欧美卡通风格”，可以把它理解为一种强调层级、对比、形状趣味和可触达感的 UI 语言：颜色更鲜明，形状更圆润，按钮更厚更像可点击实体，图标更看重轮廓识别，背景更愿意退后服务于前景信息。这不是单一官方规范，而是把 Material 对鲜明色彩与对比形状的表达性要求、Apple/Fluent 对层级与可读性的要求，转译到游戏 UI 里的一个实务范式。

注意事项与优化建议
从版权与可复制边界来看，最重要的不是“完全不能像同类型产品”，而是“不能复制别人的原创表达”。美国版权局明确指出，原创的图形与视觉作品受版权保护，而原创性可以体现在题材选择、构图、描绘方式以及设计元素的具体组合上；WIPO 也说明，电子游戏中的原创美术表达受版权保护，但某些为特定类型作品所必需的常见元素可能属于“scènes à faire”，也就是类型惯例本身不当然受保护。套用到你的 Skill 上，正确做法是：可以借鉴“木质面板、宝箱奖励、钻石货币、厚按钮”这类通用类型语言，但不要复制独特角色脸、专有 Logo、标题字体、专属图标形态、精确版式比例、边框纹样和可被识别为特定作品的装饰组合。以下是设计侧的通用注意事项，不替代正式法律意见。
